Introduction

ClaudeInOne is a production-grade framework for Claude Code. It installs 213 real skill guides, 37 specialist agents, and 95 namespaced /co: commands into any project — giving Claude expert-level knowledge across every major framework, pattern, and deployment strategy.

ℹ

Claude Code built-ins (/help, /clear, /init, /memory...) are unchanged. /co: prefix = ClaudeInOne.

Installation

Install the CLI globally via npm, then run co init inside any project:

npm install -g claudeinone
cd your-project
co init

Your First Command

Open Claude Code in your project, then try any /co: command. Claude will read the matching skill and apply production patterns automatically:

~ % claude
Welcome to Claude Code!

> /co:bootstrap-react
[Claude reading skill: frontend-react]
I can help bootstrap a production React app. I will set up Vite, Tailwind v4, and Lucide icons...

What Gets Installed

ClaudeInOne creates a .claude directory in your project root with specialized agent files and markdown skill guides.

project-root/
└── .claude/
    ├── .ck.json                # Project config
    ├── agents/                 # System prompts
    │   ├── ui-ux-designer.md
    │   └── senior-backend.md
    ├── commands/               # Slash commands
    │   ├── plan/
    │   ├── build/
    │   └── fix/
    └── skills/                 # Knowledge base
        ├── frontend-react.md
        └── database-postgres.md

Commands Reference

All ClaudeInOne commands are namespaced under /co: to distinguish them from Claude Code built-ins like /help, /clear, and /init.

/co:bootstrap
5-phase autonomous creation: Requirements → Research → Plan → Auto-generate → Validate.
/co:cook [feature]
7-step pipeline: Planning → Scout → Implementation → Testing → Code Review → Docs → Summary.
/co:fix-hard [issue]
Multi-agent debug pipeline. Scout → Debugger → Researcher → Planner → Implementation → Tester.
/co:plan-parallel
Parallel researchers and implementation planners running asynchronously to map larger complex refactors.

Example Workflows

See how ClaudeInOne commands compound to build full features.

Bootstrapping

1. /co:plan-architecture
Designs the frontend/backend divide.
2. /co:bootstrap-auth
Sets up NextAuth with proper session handles.
3. /co:bootstrap-db
Initializes Prisma and Turso config.

Production Debugging

1. /co:fix-hard [issue]
Runs root cause analysis.
2. /co:review-security
Ensures the fix didn't introduce vulnerabilities.
3. /co:write-tests
Autogenerates regression tests.
