Docs / Getting Started / Introduction

Introduction

ClaudeInOne is a production-grade framework for Claude Code. It installs 213 real skill guides, 37 specialist agents, and 95 namespaced /co: commands into any project — giving Claude expert-level knowledge across every major framework, pattern, and deployment strategy.

i

Claude Code built-ins (/help, /clear, /init, /memory...) are unchanged. /co: prefix = ClaudeInOne.

Installation

Install the CLI globally, then run co init inside any project:

npm install -g claudeinone-cli
cd your-project
co init

Your First Command

Open Claude Code in your project, then type any /co: command. Claude reads the matching skill and applies production patterns automatically:

> /co:cook add Stripe billing with subscription plans
  Claude reads: saas-billing.md, auth-session.md
  Writing app/api/checkout/route.ts ...
  Writing app/api/webhooks/stripe/route.ts ...
  * Checkout session with metadata
  * Webhook handler with signature verification
  * Customer portal route

What Gets Installed

ClaudeInOne creates a .claude directory in your project root with skills, agents, and commands.

project-root/
├── CLAUDE.md                    # Master system prompt
├── .claude/
│   ├── .ck.json                 # Version metadata
│   ├── skills/                  # 213 production skill guides
│   ├── agents/                  # 37 specialist agents
│   ├── commands/co/             # 95 /co: slash commands
│   └── settings.json
├── docs/
├── plans/
└── journals/

Example Workflows

New Feature

1. /co:plan add user auth
Creates a detailed implementation plan.
2. /co:cook add user auth
Implements end-to-end using relevant skills.
3. /co:test-gen
Generates tests for the new code.

Production Debug

1. /co:fix-hard [issue]
Deep root cause analysis.
2. /co:review-security
Checks the fix didn't introduce vulnerabilities.
3. /co:git-cm
