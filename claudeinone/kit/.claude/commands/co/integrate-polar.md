# Integrate Polar

## Invocation
`/integrate:polar [requirements]`

## Purpose
Implement Polar billing end-to-end with secure webhook handling.

## Agent Routing
- integration-specialist
- backend-developer
- frontend-developer
- tester

## Execution Protocol
1. Model products/plans and billing flows.
2. Implement API integration and webhook verification.
3. Build pricing/checkout/subscription UI.
4. Add tests and failure handling.

## Validation
- Verify assumptions before changing files.
- Run the smallest relevant checks first, then broader checks when risk is high.
- Report residual risks explicitly.

## Output Format
1. Integration plan
2. Implemented endpoints/UI
3. Test coverage summary
