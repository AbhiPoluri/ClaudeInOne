# Seed Data

## Invocation
`/seed`

## Purpose
Generate realistic seed data for local/dev testing.

## Agent Routing
- database-admin
- backend-developer

## Execution Protocol
1. Inspect schema relationships.
2. Generate representative fixtures with constraints.
3. Add repeatable seed scripts.

## Validation
- Confirm scope and constraints before making changes.
- Run relevant checks for each touched surface.
- Report unresolved risks or skipped validations.

## Output Format
1. Seed scripts/data
2. Run instructions
